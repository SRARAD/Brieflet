Brieflet Release Notes

Changes in Release 1.05

  • Fixed Screen Orientation Error for Android 4.0 on phones like the Galaxy Nexus
  • Added action to create on-slide QR codes for URLs

Changes in Release 1.04

  • Fix Force Close Error on Honeycomb 3.2 -- Due To Changes in Fragment API

Changes in Release 1.03

  • Presentation list and slide grid now use the contextual action bar and include multi-select operations.
  • Slide edit operations now fast by deferring repacking of presentations into archives.
  • Added explicit "Pack Presentations" option to repack all edits into presentation archives.
  • Added new transition options for No Transition and new Spin transition.
  • Added optional display of transitions in slide grid.
  • New option to duplicate a presentation.
  • Increased speed of building presentation list.
  • Downscaling of custom imported presentation graphics to no larger than screen dimensions.
  • Better automatic title extraction from file URIs.
  • Increased help font size, added pinch zoom.
  • Enhanced presentation details dialog.
  • Fling gestures can now be used to move to the previous or next slide in presentations.
  • Fixed issue where slide thumbnails were not being reclaimed on switching presentations.

Changes in Release 1.02

  • Intra-presentation slide links now handled as quickly as any slide-to-slide moves and include the appropriate transition effect.
  • Added aspect preservation option when importing presentations with dimensions which do not match the tablet's resolution. The user is given the option of adding sidebars.
  • Added "Add Sidebars" choice to presentations which analyzes slide dimensions and gives the option to add sidebars if desired.
  • Added Zamzar™ support for importing presentations directly from any file format handled by Zamzar. See the file conversion section for details.
  • Added Dropbox support for sending Brieflet presentations (and the converter) to Dropbox folders and for importing Brieflet presentations from Dropbox.
  • Slides display immediately for large presentations while thumbnail loading continues in the background.
  • Improved thumbnails for non-standard slide sizes to scale as the slide viewer does.
  • Added application icons to the launch application dialog.
  • Fixed issue where slide links moving from presentation to presentation kept music from the previous presentation playing.
  • Fixed sample presentation to only copy out once (in this version) to ensure all users have it and that it can be deleted.
  • Fixed an issue where Refresh was not refreshing the presentation list.
  • Minor changes to Converter.

Changes in Release 1.01

  • Added new Song Action which allows audio tracks to be added to slides (including album cover art as a thumbnail)
  • Fixed issue with copy/cut/paste in user-generated custom zip files with subdirectories.
  • Fixed issue with presentation remote which caused presentation restart key to confuse Brieflet in slide sequence.
  • Fixed an issue where the /sdcard/Slides directory was not created before attempting to write the sample presentation.
  • Show Actions shows blue action borders now while video is playing.
